Key Database Features Implemented

DDL Constraints

Uses strict PRIMARY KEY, UNIQUE, and FOREIGN KEY constraints to enforce relational integrity across users, donors, requests, and matches.

Assertions & Checks

Implements check constraints to ensure logical integrity, such as preventing a single match from linking to both blood and organ requests.

Database Triggers

An AFTER INSERT trigger automatic audit-logs every successfully created match into an immutable audit_log table.

Stored Procedures

Calculates medical blood type compatibility rules inside a pre-compiled stored procedure, matching patients with donors locally.
